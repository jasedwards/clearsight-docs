Custom Schematics

These are schematics created by devs here at Riskonnect. They include not only app generation but generating pages for common scenarios

generating an app

ng g @schematics/clearsight:cs-app My.App

Arguments:

Argument Alias Default Value Allowed Values Description
argv none none any The name of the app to generate. must be in for Storms.App or storms-app
--prefix -p cs any the component prefix for this app. defaults to cs but should be something else

This command will generate 'my-app' with component prefix of 'ap'. The prefix will always be first two letters of last word unless specified.

while the style type defaults to scss a full list of parameters can be found here

Once the app is generated you should go to index.html and replace

this:

<meta name="viewport" content="width=device-width, initial-scale=1">

with this:

<meta content='width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=0' name='viewport' />

You will then go to app-loading.html and copy the contents into your index.html between your app component selector. Once that is copied you will replace instances of with with the rgb value of your app color (i.e. 1,2,3) and instances of with your app color (i.e. #267aae)

To create a feature library

In most common scenarios, we need to build feature library around particular domain and package which includes manager page, folder page, folder manager page, new entity page and other related modules to that feature.

You can use cs-feature scematics for this purpose with different available options mentioned below which gives enough flexibility to include modules on required basis.

ng g @schematics/clearsight:cs-feature MyFeature --domainName `Domain.Name` --package `PackageName`

HINT: use '--dry-run' option in case you just want to check the file > structure supposed to be emitted by the command instead of actual file > generation.

Arguments:

Argument Alias Default Value Allowed Values Description
argv none none any The name of the feature (i.e. Claim)
--domainName -dn none any The domain name
--package -pkg none any The package name
--headerLabel -hl none any The dictionary key to use as the header of the page
--hasManager none true true/false If manager module should be included in the feature library
--hasFolder none true true/false If Folder module should be included in the feature library
--hasNewEntity none false true/false If New Entity module should be included in the feature library
--hasViews none true true/false If view lookup should be included in manger. This flag will be considered only if hasManager flag is selected
--hasSearch none true true/false If search component should be included in manger. This flag will be considered only if hasManager flag is selected

Apart from these flags the cs-feature schematic asks a question about the type of manager this feature would have. The available options are:

Manager Type selection:

Option Description
No Manager Schematic doesn't create any manager
Default Manager Creates a default manager
Domain based Split Manager Creates a domain based split manager
Default Split Manager Creates a split manager
Folder Manager (New) Creates a folder manager page
Folder Manager and Default Manager (New) Creates a folder manager and default manager page

This command will create new library under clearsight app.

To connect a library to an app

Add Routes to your app.module.ts to load your library feature:

export const MY_ROUTES: Routes = [
  {
    path: 'yourpath',
    loadChildren: () => import('@clearsight-apps/my-feature').then(m => m.MyFeatureModule)
  }
];

And then pass in the routes to RouterModule.forRoot:

RouterModule.forRoot(MY_ROUTES, { initialNavigation: 'disabled' })

To create a left hand nav

Add a nav-routing-nav instance to your app.component.html:

<nav-routing-nav [routes]="navCommands"></nav-routing-nav>

Then in your app.component.ts create your navCommands array:

navCommands: NavCommand[] = [
    { icon: 'cs-icon-home', text: 'STORMS_LBL_HOME', id: '/' },
    { icon: 'cs-icon-your-icon', text: 'YOUR_NAV_ITEM_LABEL', id: '/yourpath' }
  ];

To pass routes up to Storms.Wrapper

Add AppNavigationModule to your app.module.ts and duplicate the same navCommands array used for the nav-routing-nav component.

NOTE: this will be refactored later to remove duplication.

AppNavigationModule.forRoot([
  { icon: 'cs-icon-home', text: 'STORMS_LBL_HOME', id: '/' },
  { icon: 'cs-icon-your-icon', text: 'YOUR_NAV_ITEM_LABEL', id: '/yourpath' }
]),

To create a manager page in an app

ng g @schematics/clearsight:cs-manager MyManager --project 'app-name' --domainName `Domain.Name` --package `PackageName` --headerLabel `Dictionary Key`

Arguments:

Argument Alias Default Value Allowed Values Description
argv none none any The name of the manager (i.e. Claim)
--domainName -dn none any The domain name
--package -pkg none any The package name
--headerLabel -hl none any The dictionary key to use as the header of the page
--hasViews none true true/false If view lookup should be included in manger
--hasSearch none true true/false If search component should be included in manger
--routing (optional) none none true/false If routing should be included. If not the manager component is exported

This will actually generate a module, component and service needed for a manager page.

Filtering logic for the folder managers

Common approaches to filter the folder manager based on the parent's properties are:

Approach #1: Use searchParms property in the GridSearch

Use parentKey, parentField, parentPackageName, parentDomainName properties in the component html when the folder manager is added as a page to your entity folder

    <ta-tasks-examplefolder-manager
        [parentKey]="folderInfo.Key"
        parentDomainName="STARS.Claim"
        parentPackageName="RMIS"
        [groupKeys]="folderInfo.GroupKeys"
    >
    </ta-tasks-examplefolder-manager>

Approach #2: Use attachedEntityDomain, attachedEntityKey attributes

When there are some extra parameters to be sent to the GridSearch api then override the getExtraParamsForSearch method in the folder-manager's effect class

    protected getExtraParamsForSearch(state: IFolderPageState) {
        return {
            attachedEntityDomain: state.ParentDomainName,
            attachedEntityKey: state.ParentKey
        };
    }

Commands to know

To add a control to the control library:

Each control should have it's own module. this will allow the consumer to just add one control instead of the entire library. When adding a new control you should follow these steps.

We will assume that your control name is MyControl

ng g module MyControl --app=controls
ng g component MyControl --app=controls

The first command creates a MyControlModule file. The second command adds a MyControlComponent to that folder. After that go to controls/controls.module.ts and add your new module to CONTROLS_MODULE array. this will also allow the user to pull in all of the input controls

To create a component in an app (controls-demo...)

ng g c MyComponent --app controls-demo
