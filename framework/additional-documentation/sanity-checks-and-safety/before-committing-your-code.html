Clearsight Apps Framework

Before committing your code

These are tasks you should run before committing your code:

npm run test-<your app or lib>

This will run tests and ensure you do not break any code.

npm run format

Ensure no merge defects. The format task will format the code according to the editorconfig file. This makes for cleaner code merges

If you changed a library:

npm run build_all

This will build all the apps in production mode. This ensures there are no compile errors

If you just changed code in your application

npm run build-<your app>

This will ensure you have no compile errors in your application.

A note on ensuring no compile errors

As a developer you are able to get away with things running in dev mode (ng serve) that will throw errors when building in prod mode. This is why it is so important to make sure your code builds in production mode instead of just running ng serve all the time.
